Unicode reference

Blank Unicode character reference

Use this reference when you need to choose between visible-width blanks, zero-width separators, joiners, and word-join behavior.

U+200B Zero Width Space

Best first test for compact invisible breaks in bios, captions, and pasted text.

U+2800 Braille Pattern Blank

Blank-looking character with width. Useful when a target field needs a visible spacing position.

U+3164 Hangul Filler

Common fallback for names and messages when normal whitespace is stripped.

U+2060 Word Joiner

Prevents line breaks without adding visible width, but it can affect editing and search.

U+200C Zero Width Non-Joiner

Controls joining in scripts that use connected letters. Avoid it for casual blank spacing.

U+200D Zero Width Joiner

Joins emoji or script sequences and can change rendering. Use it only when joining is intended.

Troubleshooting

If blank text fails

It disappears

The app trimmed or normalized the mark. Try U+2800 or place the mark between visible text.

It shows a square

The font cannot render the character as blank. Try U+200B or U+3164.

It breaks layout

Directional marks can affect following text. Use U+200B for neutral spacing.

FAQ

Blank text FAQ

Is blank text the same as a normal space?

No. Blank text here means Unicode characters that can look empty but still occupy a text position.

Why does an app remove my blank text?

The app may trim whitespace, normalize Unicode, or block hidden marks in that field.

Which platform tool should I open next?

Use the WhatsApp, Discord, Instagram, TikTok, or name tool when you need platform specific steps.

Can I check what I copied?

Yes. Paste the result into the homepage detector to reveal code points and hidden marks.

Is this private?

Copying runs in your browser. You still decide where to paste the blank text afterward.
