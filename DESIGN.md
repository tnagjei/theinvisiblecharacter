# TheInvisibleCharacter · 精密索引设计系统

## 1. 视觉主题与氛围

这是一个明亮、平面、可校验的 Unicode 工具站，不是 SaaS 卡片墙。页面像一张技术索引表：暖纸底、深色分隔线、朱红色作为唯一主行动色。每个页面在三秒内必须让用户看见页面用途和下一步动作。

功能契约：用户可以在首屏开始复制、生成或检查不可见字符；说明内容不抢占工具入口。

## 2. 色板与角色

| 角色 | 色值 | 用途 |
| --- | --- | --- |
| Paper | `#f2eee4` | 页面底色 |
| Paper deep | `#e4dfd3` | 次级区域、代码格 |
| Ink | `#20201d` | 标题、边框、深色页脚 |
| Muted | `#747068` | 次要说明 |
| Rule | `#aaa499` | 细分隔线 |
| Signal | `#ce3e29` | 复制、主操作、焦点提示 |
| Signal dark | `#aa2e1d` | 主操作悬停与按下 |
| Information | `#0f62fe` | 仅用于信息状态和正文链接 |

暗色模式使用 `#20201d` 作底色，正文为 `rgba(255,255,255,.85)`，次要文字不低于 `.68`。

## 3. 排版规则

- UI 与英文正文使用 Archivo，中文回退到 `PingFang SC`、`Microsoft YaHei` 等系统字体。
- Unicode 码位、状态和短标签使用 IBM Plex Mono，启用表格数字。
- H1 左对齐，`clamp(2.35rem, 6vw, 5.8rem)`，最多三行。
- 正文基准 16px，行高 1.65。中文正文不低于 14px，行高不低于 1.5。
- 不使用斜体、渐变文字或负字距中文。

## 4. 组件样式

- 主按钮为朱红实底、直角、最小高度 48px，文案不换行。
- 次按钮为透明纸底、深色 2px 边框，悬停改变底色与文字，不改变布局。
- 输入框采用浅色底与 2px 底边框，聚焦时改为朱红色，避免无意义的全描边卡片。
- 工具条目默认以细线分组；需要强调时使用整块底色和全边框，禁止左侧竖线。
- 状态标签使用等宽字和细边框，不显示常驻“已加载”类系统文案。

## 5. 布局原则

- 基础间距按 8px 网格取值，页面容器最大宽度 1280px。
- 桌面首页和工具页使用 7:5 非对称网格，工具在左，实时索引或说明在右。
- 博客和说明页采用宽内容栏加窄侧栏的编辑式结构；移动端统一收为单列。
- 区块默认用分隔线、底色层级和留白分组，不用满屏圆角卡片。

## 6. 深度层级

- 普通内容为平面纸底。
- 可交互工具区域使用 `2px` 深色边框和浅纸底。
- 浮层、语言菜单和移动菜单使用一层暖色染色阴影：`0 14px 36px rgba(32,32,29,.16)`。
- 不使用玻璃拟态、霓虹外发光或大面积渐变。

## 7. Do 与 Don'ts

- Do：左对齐标题、显式 Unicode 码位、细密分隔线、一个清楚的朱红主动作。
- Do：让复制和检测结果立即可见，保留现有功能、链接与 SEO 文案。
- Don't：三等分卡片墙、居中 Hero、紫蓝渐变、左侧竖线强调、斜体、`transition: all`。
- Don't：用视觉改动掩盖工具错误或把说明文案放在工具之前。

## 8. 响应式行为

- `768px` 以下所有双栏、侧栏和复杂工具布局收为单列。
- `430px` 与 `375px` 下，页面左右内边距为 16px，主要触控目标最小为 44px。
- 移动端只使用顶部菜单，不注入固定底部导航，避免遮挡工具和输入框。
- 长标题、码位和 URL 必须可换行或横向滚动，页面本身不得横向溢出。

## 9. Motion 哲学

- 用户在预览中设置的动效强度为 8/10，但高频复制、输入与筛选仍保持即时。
- 只对菜单、焦点、按钮按压与首屏内容做 120–240ms 的 `transform` / `opacity` 反馈。
- 缓动使用 `cubic-bezier(0.23, 1, 0.32, 1)`；触屏 hover 效果必须隔离。
- 提供 `prefers-reduced-motion` 降级，不动画宽高、边距、位置等布局属性。

## 设计 DNA 与工艺密度

本方案借用 IBM Carbon 的 8px 网格、48px 交互高度、平面分层和等宽技术标签；借用 Wise 的暖白底、近黑文字和环形边界的克制深度，但不复制其蓝色或绿色品牌外观。

全站必须具备：低对比纸纹氛围层、品牌 `::selection`、朱红焦点环、按钮按压反馈、暖色染色阴影、等宽码位标签、流体标题和窄滚动条。它们服务于“可校验的字符索引”这一记忆点，而不是制造装饰噪声。
